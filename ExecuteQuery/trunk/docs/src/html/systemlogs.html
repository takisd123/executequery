Application Logging

Execute Query logging includes system output for information, error or debugging purposes and can be configured from the application preferences.

Generated application logs can be found within the directory: ~/.executequery/logs (~ - the user's home directory)

Main Application Log

The primary system output log is the file system.log. The application output console, typically located as a docked component at the bottom of the application window (select View > Output Console if not currently visible) mirrors the output written to this log file.

Please use the contents of the log file when reporting any application errors and/or bugs.

Data Import and Export Logging

During any data import/export action, the output as generated and displayed within the final step of respective import/export processes is written to the log files import.log and export.log.

The above include all relevant information regarding the process including what was imported/exported, to where, the duration of the process and any success or error messages as may have been generated during each individual process.

Viewing System Logs

System logs may be accessed and viewed in their entirety from the menu item Tools > System Logs > <log file type>.

The above will open respective system logs within a simple text viewer as a part of the central pane. Each log may be cleared (contents deleted and the log reset) or reloaded as required.

Use the log selection drop-down box to display different logs from the one viewer.

Logs may also be reset using the appropriate menu item within the menu Tools > System Logs > Reset Logs > <log file type>
