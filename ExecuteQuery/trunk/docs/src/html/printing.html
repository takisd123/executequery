Printing

Printing is available from most of Execute Query's open table displays, diagrams (ERD) and text components and may be selected from the main toolbar's Print button or from the main menu item File > Print. A print preview provides you with some idea as to how the view will be printed.

Selecting to print from the Query Editor provides for both printing the results table from an executed query or the SQL text area.

All windows with significant text area components such as the scratch pad and system logs support printing.

Where a dialog or central pane generates SQL text (ie. Create Table, Create Index etc dialogs), the generated text becomes the printable component.

Table and schema descriptions (columns, data types etc.) may be printed from the Database Browser pane.

Refer to the help documents of respective components for further information on printing specific views.
