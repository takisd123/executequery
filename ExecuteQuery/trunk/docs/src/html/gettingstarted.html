Getting Started
An application GUI framework overview is provided below.

Application GUI Overview

Execute Query employs the now common tabbed and dockable components interface for such tools and indeed most IDEs.

The central component view is the primary work area providing the placement of all editor and browsing components including the Query Editor, the Database Object Browser and ERD generator, amongst others. The placement of the central components is fixed and these can not be maneuvered to the left, right or bottom docking locations.

The left (top and bottom), bottom (center) and right (top and bottom) panes provide for the other dockable components such as the connections tree or the installed database drivers list. All these components may be dragged to any of the three positions as desired. Other dockable components other than the standard available on initial application startup are available from the View menu and include the System Properties, SQL Key Words and SQL State Codes, each of which are described elsewhere in this document.

The Execute Query toolbar is also configurable via the system preferences. Individual toolbars may be moved horizontally or vertically along the top of the application window allowing users to reorder the tool bar components or to extend the tool bars downward.

Central Scrolling Tab Components

The central component tabs may be scrolled left and right using the scroll left-right buttons provided on the right-hand side of the tabs.

A list of currently open tabs is available by selecting the down-arrow button to the right of the left-right scrolling buttons. Selecting this generates a menu of all the currently open central tab components. Selecting any one of these shall scroll the tabs to the selection and display the component as the focused view.
